# @pickle-pee/config

## 职责

配置与资源管理。管理配置合并顺序（默认 → 全局 → 项目 → 会话 → 命令行）、资源目录定位、技能/模板/上下文文件的加载规范。

## 导出

- 配置合并接口
- 资源定位接口
- 配置层级类型
- 技能与模板加载类型

## 依赖方向

- 依赖: 无（本项目叶子节点）
- 被依赖: `app-extensions`, `app-cli`

## 禁止事项

- 不包含运行时逻辑
- 不实现工具
- 不管理会话状态

## 内部结构

- `domain/` — 配置模型、资源路径定义
- `services/` — 配置加载与合并
- `types/` — 包级公共类型定义

## 验证

- `npx tsc --noEmit` 类型检查通过
